01 · Introducción

Visión general del proceso

El sistema DRAXTON RFQ Job Executor automatiza el flujo de información desde el ERP Expertis hasta los documentos de análisis de costes. El único dato que el usuario introduce manualmente es el RFQ Number y la Versión.

ERP

Expertis — Fuente de verdad

Toda la información comercial de la oferta: cliente, referencia, volúmenes de producción, responsable logístico. El sistema se conecta automáticamente al introducir el RFQ.

tbOfertaComercialCabecera vIPSCIRampUps tbMaestroEmpresa
TQP

TQP — Parámetros técnicos de cotización

Se rellena el Excel TQP con datos del ERP. El usuario completa los datos técnicos (pesos, velocidades, machos) y el sistema los recoge y guarda en base de datos.

TQP Data Insert TQP Brakes Data Recolection RFQ_TQP_DataCastingCost
CAST

Casting Cost — Costes de fundición

Se crea el Excel de costes con datos de Expertis + TQP pre-cargados. El usuario completa los costes por planta y el Recolect los persiste en 7 tablas de base de datos.

Casting Data Insert Casting Data Recolect 8 plantas de fundición
MACH

Machining Cost — Costes de mecanizado

Proceso paralelo al Casting pero independiente: un archivo Excel por planta, relleno directo desde Expertis sin necesidad de TQP. 3 plantas de mecanizado.

Machining Data Insert Machining Data Recolect Lleida · Wroclaw · Wuhu
PA

PA Data Insert — Controlling

Paso final. Toma los costes guardados (Casting o Machining) + volúmenes de Expertis y los consolida en la PlantillaPA para análisis financiero de Controlling.

PA Data Insert PlantillaPA.xlsx Database A → J

Expertis es la única fuente de entrada que el sistema necesita del usuario. El resto del proceso es automático. Se consultan tres tablas/vistas para nutrir todos los módulos.

03 · Persistencia

Tablas de base de datos

El sistema usa SQL Server como almacén intermedio. Los datos pasan de Excel a BD (Recolect) para que los módulos posteriores puedan consumirlos sin depender de archivos.

TablaMóduloContenidoClave única

RFQ_CastingCost_Fund_GeneralInfo Casting Info general: cavidades, pesos, rendimiento, scrap, precio base FK a Header
RFQ_CastingCost_Fund_MaterialCost Casting Costes de materiales fila a fila (material, unidades, coste, totales) FK a Header + row_number
RFQ_CastingCost_Fund_Totals Casting Subtotales por grupo de materiales FK a Header + row_number
RFQ_CastingCost_Fund_TotalsGlobal Casting Totales globales (por kilo y por pieza) FK a Header + row_number
RFQ_CastingCost_Fund_InfoTechnicalStudy Casting Estudio técnico: piezas/hora y operarios por fila FK a Header + row_number
RFQ_CastingCost_Fund_InfoPA Casting / Machining Info PA: materiales y coste EUR (+ RMB para China) FK a Header + row_number
RFQ_MachiningCost_Inversion Machining Datos de inversión: Need True, Client Not Paid, Paid Client + breakdown 6 años. Additional Facilities solo para Wuhu. FK a Header

09 · Referencia

Archivos generados

Todos los archivos siguen una nomenclatura con RFQ, versión, planta e identificador de estudio para asegurar trazabilidad completa.

TipoCarpetaNombre del archivoNotas
TQP Ingenieria Output/TQP/ TQP.xlsm Archivo con macros VBA
Casting base Ingenieria Output/Casting/ CostesIngenieria_RFQ{n}_v{v}.xlsx Se consume al crear el estudio
Casting estudio Ingenieria Output/Casting/ CostesIngenieria_RFQ{n}_v{v}_{PLANTA}_study{sv}.xlsx Copia de trabajo por planta
Machining base Ingenieria Output/Machining/{PLANTA}/ Cost_Machining_RFQ{n}_v{v}_{PLANTA}.xlsx Se consume al crear el estudio
Machining estudio Ingenieria Output/Machining/{PLANTA}/ Cost_Machining_RFQ{n}_v{v}_{PLANTA}_sv{sv}.xlsx Copia de trabajo por planta
PlantillaPA Controlling Output/ PlantillaPA_RFQ{n}_v{v}_ctrl{cv}.xlsx Se actualiza incrementalmente
♻️
Archivos base vs. archivos de estudio: Los archivos base se crean durante el Data Insert y se consumen (renombran) en el primer Recolect. Para crear un nuevo estudio con la misma versión RFQ, es necesario volver a ejecutar el Data Insert para regenerar el archivo base. El archivo PlantillaPA es la excepción: si existe, se actualiza sin consumirse.
